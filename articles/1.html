react 生命周期

  • Mounting

    constructor -》 static getDerivedStateFromProps -》render -》 componentDidMount

  • Updating

    static getDerivedStateFromProps -> shouldComponentUpdate -> render -> getSnapshotBeforeUpdate ->componentDidUpdate

    • static getDerivedStateFromProps(props, state)

      getDerivedStateFromProps is invoked right before calling the render method, both on the initial mount and on subsequent updates. It should return an object to update the state, or null to update nothing.

      更新是通过 merge 的方式进行的

    • getSnapshotBeforeUpdate (prevProps, prevState)

      getSnapshotBeforeUpdate(prevProps, prevState) is invoked right before the most recently rendered output is committed to e.g. the DOM. It enables your component to capture some information from the DOM (e.g. scroll position) before it is potentially changed. Any value returned by this lifecycle will be passed as a parameter to componentDidUpdate().

      getSnapshotBeforeUpdate(prevProps, prevState) 在最近的一次 render 被挂载到dom节点之前被调用，这允许你的组件在dom节点在潜在的改变前获取到dom节点的信息。这个函数返回的值会传递给componentDidUpdate(prevProps, prevState, snapshot)的第三个参数。

2.基础

  1. 主入口：main函数

  2. 一切皆对象，任何能保存在变量中的东西都是对象，任何对象都是class的实例，任何类都是继承自 Object class

  3. 强类型，类型声明可选，因为dart有类型推断，变量类型取决与初始化的值。

  4. 支持泛型，比如List<int>

  5. 支持高阶函数，函数可以被绑定到类或对象上，也可以在函数内定义函数。

  6. Dart supports top-level variables, as well as variables tied to a class or object (static and instance variables). Instance variables are sometimes known as fields or properties.

  7. 没有显式的访问控制声明，如 public、protected等，下划线开头的属性是私有的。

  8. 标识符以字母下划线开头，后面随意

  9. 表达式语句与声明语句很不同

  10. dart工具回报两种问题，error 和 warning，错误包括运行时错误和编译时错误

runes

In Dart, runes are the UTF-32 code points of a string.

Unicode defines a unique numeric value for each letter, digit, and symbol used in all of the world’s writing systems. Because a Dart string is a sequence of UTF-16 code units, expressing 32-bit Unicode values within a string requires special syntax.

main() {
  var clapping = '\u{1f44f}';
  print(clapping);
  print(clapping.codeUnits);
  print(clapping.runes.toList());

  Runes input = new Runes(
      '\u2665  \u{1f605}  \u{1f60e}  \u{1f47b}  \u{1f596}  \u{1f44d}');
  print(new String.fromCharCodes(input));
}

flow语法

书写步骤：

  • 书写操作块
  • 书写控制流程

操作块

st=>start: 开始

st 是变量名，start是操作模块，：后面是要在操作块里面显示的文字

流程控制

# 定义操作块
st=>start: 开始
e=>end: 结束
op1=>operation: 第一个操作块
op2=>operation: 第二个操作块
cond1=>condition: 第一个判断
# 定义控制流程
st->op1->cond1(yes，left)->e
cond1(no)->op2->op1
