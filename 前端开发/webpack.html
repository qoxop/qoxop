webpack

author: qoxop update at: 2019-03-23

入口

单入口

//简写
const config = {
  entry: './path/to/my/entry/file.js'
};
module.exports = config
//对象写法
const config = {
  entry: {
    main: './path/to/my/entry/file.js'
  }
};

多入口（多页面）

//告诉webpack 需要构建 3 个独立分离的依赖图
const config = {
  entry: {
    pageOne: './path/to/my/one.js',
    pageTwo: './path/to/my/two.js'
  }
};

输出

注意 ：即使可以存在多个入口起点，但只指定一个输出配置

//创建单个chunk
const config = {
  output: {
    filename:'bundle.js'	//输出文件名
    path:'/home/proj/public/assets'	//输出路径
  }
}
//创建多个chunk
{
  entry: {
    app: './src/app.js',
    search: './src/search.js'
  },
  output: {
    filename: '[name].js',	//[name]代表入口文件名称
    path: __dirname + '/dist'	//__dirname魔术变量
  }
}

loader

用于对模块的源代码进行转换。处理时机是 import 或加载模块时

//简写模式
module.exports = {
  module: {
    rules: [
      { test: /\.css$/, use: 'css-loader' },//test文件匹配规则 
      { test: /\.ts$/, use: 'ts-loader' }	//use使用的loader插件 ，字符串或对象数组
    ]
  }
};
//详细模式
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
          { loader: 'style-loader' },
          {
            loader: 'css-loader',
            options: {
              modules: true
            }
          }
        ]
      }
    ]
  }
  • 支持链式传递（给use属性传递一个数组），前一个loader将处理结果传递给下个loader处理
  • loader支持同步或异步
  • 能接受查询参数？？
  • 可以通过options对象进行配置
  • loader能够产生额外的任意文件
